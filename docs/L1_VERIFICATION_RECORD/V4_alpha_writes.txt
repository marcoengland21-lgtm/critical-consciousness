Interpretation:
- The INSERT was rejected by Postgres RLS at the database layer.
- Error code 42501 ("insufficient_privilege") is the standard Postgres
  signal for an RLS WITH CHECK violation.
- This is the EXPECTED PASS. RLS structurally prevents user_alpha (a member
  of test-group only) from writing rows scoped to Watermelon, even though
  alpha set author_id = own UUID (so the only blocker is the group_id
  membership check, which is exactly the L1 contract).
- Combined with the symmetric beta-into-test-group rejection captured in
  V4_beta_writes.txt, both directions of cross-group write are blocked.
